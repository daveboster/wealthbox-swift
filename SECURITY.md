# Security Policy

## Reporting a Vulnerability

Do not open a public issue for security-sensitive reports.

Report vulnerabilities through GitHub's private vulnerability reporting for this
repository when available. If private vulnerability reporting is unavailable,
contact the repository owner through GitHub and include:

- A short description of the issue.
- Affected package versions or commits.
- Reproduction steps or proof-of-concept details.
- Any known mitigation.

Do not include Wealthbox access tokens or private CRM data in public issues,
pull requests, screenshots, logs, or test fixtures.

We will acknowledge valid reports, investigate the impact, and publish a fix and
release notes when appropriate.
